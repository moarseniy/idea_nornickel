# idea_nornickel

## MVP

В репозитории реализован локальный MVP `Hypothesis Lab`: FastAPI-приложение со статическим веб-интерфейсом, SQLite-хранилищем и Docker Compose запуском.

Что уже есть:

- проекты с целью, ограничениями, доменом и командным участником;
- загрузка и парсинг `.pdf`, `.docx`, `.xlsx`, `.csv`, `.txt`, `.md`, `.png`, `.jpg`;
- импорт демонстрационных материалов из папки `data`;
- мультиязычные источники: русский, английский, китайский и другие Unicode-тексты;
- автоматическое построение графа знаний по источникам и гипотезам;
- генерация проверяемых гипотез через OpenAI API;
- экономичный постраничный разбор PDF: text-layer -> EasyOCR -> OpenAI vision только для страниц без OCR-текста;
- ранжирование по новизне, реализуемости, эффекту и риску;
- дорожная карта проверки, обоснования и привязка к источникам;
- чат для экспертной коррекции системы;
- фидбэк, статусы гипотез и журнал версий;
- экспорт результатов в JSON и CSV.

### Локальный запуск

```bash
cp .env.example .env
# добавьте OPENAI_API_KEY в .env
docker compose up --build
```

После старта интерфейс доступен по адресу:

```text
http://localhost:8080
```

Если `OPENAI_API_KEY` не задан, можно загрузить и распарсить источники, но генерация гипотез и чат вернут ошибку конфигурации.

Для `.png`/`.jpg` при наличии `OPENAI_API_KEY` выполняется vision-анализ через OpenAI Responses API: описание изображения добавляется в текст документа, участвует в графе знаний и затем попадает в контекст генерации гипотез.

PDF обрабатываются экономично:

1. Если у страницы есть редактируемый текстовый слой, извлекается только текст без OCR и vision.
2. Если текстовый слой слишком короткий или страница PDF является изображением, приложение рендерит страницу и пробует EasyOCR.
3. Если OCR не нашел достаточно текста, страница отправляется в OpenAI vision. Количество таких страниц ограничено `PDF_VISION_MAX_PAGES`.

Мультиязычность:

- обычный текстовый слой PDF, DOCX, XLSX, CSV, TXT и MD сохраняется в Unicode без перевода и без потери исходных терминов;
- OCR по умолчанию настроен на `ru,en,ch_sim`: русский, английский и упрощенный китайский;
- при загрузке или импорте источников можно выбрать язык конкретной партии файлов; пустое значение означает авто/default `ru,en,ch_sim`;
- из-за ограничений EasyOCR китайский OCR запускается отдельной совместимой группой `ch_sim,en`, а русский/английский — группой `ru,en`;
- для традиционного китайского можно добавить `ch_tra`, для других языков — расширить `PDF_OCR_LANGUAGES`;
- в metadata документа сохраняются `detected_languages`, `script_counts` и признак `multilingual`.

Логи диагностики:

```bash
docker compose logs -f hypothesis-lab
```

Минимальная проверка текущей OpenAI-конфигурации:

```bash
curl -X POST http://localhost:8080/api/openai/check
```

### Переменные окружения

- `OPENAI_API_KEY` — ключ OpenAI API.
- `OPENAI_MODEL` — модель для генерации, по умолчанию `gpt-5.2`.
- `OPENAI_BASE_URL` — опциональный совместимый endpoint; для обычного OpenAI API оставьте пустым.
- `OPENAI_GRAPH_EXTRACTION` — LLM-извлечение узлов и связей из документов, по умолчанию `true`.
- `OPENAI_VISION_DETAIL` — детализация vision-запросов: `high`, `low` или `auto`, по умолчанию `high`.
- `LOG_LEVEL` — уровень логов приложения, по умолчанию `INFO`.
- `UVICORN_LOG_LEVEL` — уровень логов Uvicorn, по умолчанию `info`.
- `MAX_DOCUMENT_CHARS` — ограничение текста одного документа после парсинга.
- `MAX_CONTEXT_CHARS` — ограничение контекста, передаваемого в модель.
- `PDF_OCR_ENABLED` — включить EasyOCR для image-only страниц PDF, по умолчанию `true`.
- `PDF_OCR_LANGUAGES` — языки EasyOCR через запятую, по умолчанию `ru,en,ch_sim`.
- `PDF_OCR_MIN_CHARS` — минимальная длина OCR-текста, после которой vision уже не нужен.
- `PDF_TEXT_LAYER_MIN_CHARS` — минимальная длина текстового слоя PDF, чтобы считать страницу полноценно распознанной без OCR.
- `PDF_RENDER_DPI` — DPI рендера PDF-страницы для OCR/vision.
- `PDF_VISION_MAX_PAGES` — максимум страниц одного PDF, которые можно отправить в vision.

### Архитектура

```text
Browser UI
  -> FastAPI
      -> Document parsers
      -> Knowledge graph builder
      -> OpenAI Responses API wrapper
      -> SQLite events/versioning store
```

Основные файлы:

- `app/main.py` — API, загрузка файлов, генерация, чат, экспорт.
- `app/openai_service.py` — интеграция с OpenAI API.
- `app/knowledge.py` — эвристическое извлечение графа из источников.
- `app/db.py` — SQLite-схема, версии, фидбэк, события.
- `app/static/` — веб-интерфейс.
- `docker-compose.yml` — локальный запуск с volume для `storage`.
