# idea_nornickel

## MVP

В репозитории реализован локальный MVP `Hypothesis Lab`: FastAPI-приложение со статическим веб-интерфейсом, SQLite-хранилищем и Docker Compose запуском.

Что уже есть:

- проекты с целью, ограничениями, доменом и командным участником;
- загрузка и парсинг `.pdf`, `.docx`, `.xlsx`, `.csv`, `.txt`, `.md`;
- импорт демонстрационных материалов из папки `data`;
- автоматическое построение графа знаний по источникам и гипотезам;
- генерация проверяемых гипотез через OpenAI API;
- ранжирование по новизне, реализуемости, эффекту и риску;
- дорожная карта проверки, обоснования и привязка к источникам;
- чат для экспертной коррекции системы;
- фидбэк, статусы гипотез и журнал версий;
- экспорт результатов в JSON и CSV.

### Локальный запуск

```bash
cp .env.example .env
# добавьте OPENAI_API_KEY в .env
docker compose up --build
```

После старта интерфейс доступен по адресу:

```text
http://localhost:8080
```

Если `OPENAI_API_KEY` не задан, можно загрузить и распарсить источники, но генерация гипотез и чат вернут ошибку конфигурации.

### Переменные окружения

- `OPENAI_API_KEY` — ключ OpenAI API.
- `OPENAI_MODEL` — модель для генерации, по умолчанию `gpt-5.2`.
- `OPENAI_GRAPH_EXTRACTION` — LLM-извлечение узлов и связей из документов, по умолчанию `true`.
- `MAX_DOCUMENT_CHARS` — ограничение текста одного документа после парсинга.
- `MAX_CONTEXT_CHARS` — ограничение контекста, передаваемого в модель.

### Архитектура

```text
Browser UI
  -> FastAPI
      -> Document parsers
      -> Knowledge graph builder
      -> OpenAI Responses API wrapper
      -> SQLite events/versioning store
```

Основные файлы:

- `app/main.py` — API, загрузка файлов, генерация, чат, экспорт.
- `app/openai_service.py` — интеграция с OpenAI API.
- `app/knowledge.py` — эвристическое извлечение графа из источников.
- `app/db.py` — SQLite-схема, версии, фидбэк, события.
- `app/static/` — веб-интерфейс.
- `docker-compose.yml` — локальный запуск с volume для `storage`.
